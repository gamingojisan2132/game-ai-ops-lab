# Agent Rules

## 基本方針

- このリポジトリは、AIエージェントが継続的にゲーム開発するための実験用リポジトリです。
- 大きな変更を一度に行わず、1タスク1目的で小さく進めてください。
- 作業前に必ず `docs/TASKS.md` を確認してください。
- 作業後に必ず `docs/TASKS.md` と必要に応じて `docs/WORKLOG.md` を更新してください。
- 仕様判断をした場合は `docs/DECISIONS.md` に記録してください。
- 人間が読む必要のある文章は日本語で書いてください。

## Windows

- Windows環境では `npm` ではなく `npm.cmd` を優先して使ってください。
- PowerShellの実行ポリシー差分は、プロジェクト問題ではなく環境問題として切り分けてください。

## 回避策

回避策を使う場合は、必ず `docs/WORKLOG.md` または `docs/TASKS.md` に以下を記録してください。

- 通常のエンジニアならどう直すか
- 今回なぜ回避策を選んだか
- 回避策のデメリット
- 将来的に戻すべきか
- 環境問題かプロジェクト問題か

特に以下は要注意です。

- `npm.cmd` への切り替え
- buildスキップ
- mock化
- fallback実装
- polyfill追加
- 無効化フラグ
- 環境変数回避
- サンドボックス回避

## Browser / Frontend

- HTTP 200だけで成功扱いにしないでください。
- 画面表示を確認してください。
- ブラウザコンソールエラーを確認してください。
- build成功だけで完了扱いにしないでください。
- 実画面確認できない場合は「未確認」と明記してください。

## 禁止事項

- 既存ファイルの大規模削除をしない。
- 認証情報、秘密鍵、トークンを作成・保存・変更しない。
- 原因不明のまま依存関係を大量追加しない。
- エラーを隠すためにテストやビルドを無効化しない。
